Deno provides a built-in cron API for scheduling recurring tasks. This is particularly useful for running periodic jobs like cleanup tasks, data synchronization, or scheduled reports.

## Deno.cron()

Schedules a cron job to run on a recurring basis.

```typescript
function cron(
  name: string,
  schedule: string | CronSchedule,
  handler: () => void | Promise<void>
): Promise<void>

function cron(
  name: string,
  schedule: string | CronSchedule,
  options: { backoffSchedule?: number[]; signal?: AbortSignal },
  handler: () => void | Promise<void>
): Promise<void>
```

<ParamField path="name" type="string" required>
  A unique name for the cron job
</ParamField>

<ParamField path="schedule" type="string | CronSchedule" required>
  The schedule for when to run the job. Can be a cron string or a schedule object
</ParamField>

<ParamField path="options.backoffSchedule" type="number[]">
  Array of milliseconds to wait between retries if the handler fails
</ParamField>

<ParamField path="options.signal" type="AbortSignal">
  An AbortSignal to stop the cron job
</ParamField>

<ParamField path="handler" type="() => void | Promise<void>" required>
  The function to execute on each scheduled run
</ParamField>

## Cron String Format

Cron strings follow the standard 5-field format:

```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
│ │ │ │ │
* * * * *
```

**Examples:**

```typescript
// Every minute
Deno.cron("every minute", "* * * * *", () => {
  console.log("Running every minute");
});

// Every hour at minute 0
Deno.cron("hourly", "0 * * * *", () => {
  console.log("Running every hour");
});

// Every day at midnight
Deno.cron("daily", "0 0 * * *", () => {
  console.log("Running daily at midnight");
});

// Every Monday at 9 AM
Deno.cron("weekly-report", "0 9 * * 1", () => {
  console.log("Generating weekly report");
});

// Every 15 minutes
Deno.cron("quarter-hourly", "*/15 * * * *", () => {
  console.log("Running every 15 minutes");
});
```

## CronSchedule Object

Instead of a cron string, you can use a schedule object for more readable configurations:

```typescript
interface CronSchedule {
  minute?: number | { exact: number | number[] } | {
    start?: number;
    end?: number;
    every?: number;
  };
  hour?: number | { exact: number | number[] } | {
    start?: number;
    end?: number;
    every?: number;
  };
  dayOfMonth?: number | { exact: number | number[] } | {
    start?: number;
    end?: number;
    every?: number;
  };
  month?: number | { exact: number | number[] } | {
    start?: number;
    end?: number;
    every?: number;
  };
  dayOfWeek?: number | { exact: number | number[] } | {
    start?: number;
    end?: number;
    every?: number;
  };
}
```

**Examples:**

```typescript
// Every 2 hours
Deno.cron("bi-hourly", { hour: { every: 2 } }, () => {
  console.log("Running every 2 hours");
});

// Every day at 3:30 PM
Deno.cron("daily-3-30pm", { hour: 15, minute: 30 }, () => {
  console.log("Running at 3:30 PM daily");
});

// Every weekday at 9 AM
Deno.cron("weekday-morning", {
  hour: 9,
  minute: 0,
  dayOfWeek: { start: 1, end: 5 } // Monday to Friday
}, () => {
  console.log("Running on weekdays at 9 AM");
});

// First day of every month at midnight
Deno.cron("monthly", {
  dayOfMonth: 1,
  hour: 0,
  minute: 0
}, () => {
  console.log("Running on the 1st of each month");
});

// Every 5 minutes during business hours
Deno.cron("business-hours", {
  minute: { every: 5 },
  hour: { start: 9, end: 17 }
}, () => {
  console.log("Running every 5 minutes during 9 AM - 5 PM");
});
```

## Error Handling and Retries

### Backoff Schedule

If a cron handler throws an error, you can configure automatic retries with exponential backoff:

```typescript
Deno.cron(
  "data-sync",
  "0 * * * *", // Every hour
  {
    backoffSchedule: [1000, 5000, 10000], // Retry after 1s, 5s, 10s
  },
  async () => {
    await syncData(); // Might throw an error
  }
);
```

If the handler fails:
1. First retry after 1 second
2. Second retry after 5 seconds
3. Third retry after 10 seconds
4. If all retries fail, the error is logged and the next scheduled run will execute normally

### Manual Error Handling

```typescript
Deno.cron("backup", "0 2 * * *", async () => {
  try {
    await performBackup();
    console.log("Backup completed successfully");
  } catch (error) {
    console.error("Backup failed:", error);
    await notifyAdmins("Backup failed: " + error.message);
    // Error is caught, cron will continue on schedule
  }
});
```

## Canceling Cron Jobs

### Using AbortSignal

```typescript
const controller = new AbortController();
const { signal } = controller;

Deno.cron(
  "temporary-job",
  "* * * * *",
  { signal },
  () => {
    console.log("This job can be cancelled");
  }
);

// Later, cancel the job
controller.abort();
```

### Conditional Cancellation

```typescript
const controller = new AbortController();
let runCount = 0;

Deno.cron(
  "limited-job",
  "*/5 * * * *",
  { signal: controller.signal },
  () => {
    runCount++;
    console.log(`Run ${runCount}`);
    
    if (runCount >= 10) {
      console.log("Job complete, canceling");
      controller.abort();
    }
  }
);
```

## Complete Examples

### Database Cleanup

```typescript
import { Database } from "./db.ts";

const db = new Database();

// Clean up old records every day at 2 AM
Deno.cron(
  "cleanup-old-records",
  "0 2 * * *",
  {
    backoffSchedule: [5000, 30000, 60000], // Retry with backoff if it fails
  },
  async () => {
    console.log("Starting cleanup...");
    
    const thirtyDaysAgo = Date.now() - (30 * 24 * 60 * 60 * 1000);
    const deleted = await db.deleteWhere(
      "created_at < ?",
      [thirtyDaysAgo]
    );
    
    console.log(`Cleanup complete: deleted ${deleted} records`);
  }
);
```

### Data Synchronization

```typescript
// Sync data from external API every 15 minutes
Deno.cron(
  "api-sync",
  { minute: { every: 15 } },
  async () => {
    console.log("Syncing data from API...");
    
    try {
      const response = await fetch("https://api.example.com/data");
      const data = await response.json();
      
      const kv = await Deno.openKv();
      await kv.set(["external_data", "latest"], {
        data,
        syncedAt: Date.now(),
      });
      kv.close();
      
      console.log("Sync completed successfully");
    } catch (error) {
      console.error("Sync failed:", error);
      throw error; // Will trigger retry with backoff
    }
  }
);
```

### Report Generation

Deno.cron(

### Cache Warming

```typescript
// Warm up cache every 5 minutes during business hours
Deno.cron(
  "cache-warmer",
  {
    minute: { every: 5 },
    hour: { start: 9, end: 17 },
    dayOfWeek: { start: 1, end: 5 }, // Monday to Friday
  },
  async () => {
    console.log("Warming cache...");
    
    const popularPages = [
      "/",
      "/products",
      "/about",
      "/contact",
    ];
    
    for (const page of popularPages) {
      try {
        const response = await fetch(`https://example.com${page}`);
        await response.text(); // Ensure full response is loaded
        console.log(`Cached: ${page}`);
      } catch (error) {
        console.error(`Failed to cache ${page}:`, error);
      }
    }
    
    console.log("Cache warming complete");
  }
);
```

### Health Checks

```typescript
// Monitor service health every minute
Deno.cron(
  "health-check",
  "* * * * *",
  async () => {
    const services = [
      { name: "API", url: "https://api.example.com/health" },
      { name: "Database", url: "https://db.example.com/ping" },
    ];
    
    for (const service of services) {
      try {
        const response = await fetch(service.url, {
          signal: AbortSignal.timeout(5000), // 5 second timeout
        });
        
        if (response.ok) {
          console.log(`${service.name}: OK`);
        } else {
          console.error(`${service.name}: Failed (${response.status})`);
          await alertOncall(`${service.name} is down`);
        }
      } catch (error) {
        console.error(`${service.name}: Error -`, error);
        await alertOncall(`${service.name} is unreachable`);
      }
    }
  }
);
```

### Multiple Cron Jobs

```typescript
// Run multiple cron jobs together
const jobs = [
  {
    name: "hourly-task",
    schedule: "0 * * * *",
    handler: () => console.log("Hourly task"),
  },
  {
    name: "daily-task",
    schedule: { hour: 0, minute: 0 },
    handler: () => console.log("Daily task"),
  },
  {
    name: "weekly-task",
    schedule: "0 0 * * 0",
    handler: () => console.log("Weekly task"),
  },
];

for (const job of jobs) {
  Deno.cron(job.name, job.schedule, job.handler);
}

console.log(`Started ${jobs.length} cron jobs`);
```

## Best Practices

1. **Use Unique Names**: Each cron job must have a unique name
2. **Handle Errors**: Always handle errors in your handlers
3. **Configure Retries**: Use `backoffSchedule` for important jobs
4. **Log Activity**: Log when jobs start and finish
5. **Set Timeouts**: Don't let jobs run indefinitely
6. **Use AbortSignal**: Allow jobs to be cancelled gracefully
7. **Avoid Overlap**: Ensure jobs don't run longer than their schedule interval
8. **Test Schedules**: Verify your cron expressions do what you expect
9. **Monitor Jobs**: Track job execution and failures
10. **Keep Handlers Small**: Offload heavy work to background tasks

## Cron Expression Quick Reference

```bash
# Every minute
* * * * *

# Every 5 minutes
*/5 * * * *

# Every hour
0 * * * *

# Every day at midnight
0 0 * * *

# Every day at 3:30 AM
30 3 * * *

# Every Monday at 9 AM
0 9 * * 1

# First day of every month
0 0 1 * *

# Every weekday at noon
0 12 * * 1-5

# Every 15 minutes during business hours (9 AM - 5 PM)
*/15 9-17 * * *
```

## Limitations

- Cron jobs run in the same process as your application
- Maximum backoff schedule: 5 intervals
- Maximum backoff interval: 1 hour (3,600,000 ms)
- Jobs must complete before the next scheduled run to avoid overlap
- Unique names are required per database/deployment

## Permissions

Cron jobs don't require special permissions, but the code they execute may need appropriate permissions:

```bash
# If cron job reads/writes files
deno run --allow-read --allow-write script.ts

# If cron job makes network requests
deno run --allow-net script.ts
```
