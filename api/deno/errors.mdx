title: Deno Error Types
description: Standard error types thrown by Deno APIs
---

Deno provides a set of standard error types that are thrown by various APIs. These errors help you handle specific failure conditions in your code.

## Deno.errors Namespace

All Deno-specific errors are available under the `Deno.errors` namespace:

```typescript
try {
  const file = await Deno.open("./nonexistent.txt");
} catch (error) {
  if (error instanceof Deno.errors.NotFound) {
    console.error("File not found");
  }
}
```

## File System Errors

### NotFound

Raised when a file or directory is not found.

```typescript
class NotFound extends Error {}
```

**Example:**

```typescript
try {
  const file = await Deno.open("./missing.txt");
} catch (error) {
  if (error instanceof Deno.errors.NotFound) {
    console.error("The file does not exist");
  }
}
```

**Common causes:**
- Reading a file that doesn't exist
- Opening a non-existent directory
- Accessing a path that has been deleted

### AlreadyExists

Raised when trying to create a resource that already exists.

```typescript
class AlreadyExists extends Error {}
```

**Example:**

```typescript
try {
  await Deno.mkdir("./existing-dir");
} catch (error) {
  if (error instanceof Deno.errors.AlreadyExists) {
    console.error("Directory already exists");
  }
}
```

### IsADirectory

Raised when trying to open, create, or write to a directory.

```typescript
class IsADirectory extends Error {}
```

**Example:**

```typescript
try {
  await Deno.readFile("./some-directory");
} catch (error) {
  if (error instanceof Deno.errors.IsADirectory) {
    console.error("Cannot read a directory as a file");
  }
}
```

### NotADirectory

Raised when trying to perform an operation on a path that is not a directory.

```typescript
class NotADirectory extends Error {}
```

**Example:**

```typescript
try {
  for await (const entry of Deno.readDir("./file.txt")) {
    console.log(entry.name);
  }
} catch (error) {
  if (error instanceof Deno.errors.NotADirectory) {
    console.error("Path is not a directory");
  }
}
```

### FilesystemLoop

Raised when too many symbolic links were encountered when resolving a filename.

```typescript
class FilesystemLoop extends Error {}
```

**Example:**

```typescript
try {
  await Deno.realPath("./circular-symlink");
} catch (error) {
  if (error instanceof Deno.errors.FilesystemLoop) {
    console.error("Circular symbolic link detected");
  }
}
```

## Permission Errors

### PermissionDenied

Raised when the operating system denies access to a resource.

```typescript
class PermissionDenied extends Error {}
```

**Example:**

```typescript
try {
  await Deno.writeFile("/root/file.txt", new Uint8Array());
} catch (error) {
  if (error instanceof Deno.errors.PermissionDenied) {
    console.error("Operating system denied access to the file");
  }
}
```

**Note:** Before Deno 2.0, this error was also raised when Deno runtime permissions were not granted. As of Deno 2.0, that case is now handled by `NotCapable`.

### NotCapable

Raised when trying to perform an operation without the required Deno permission.

```typescript
class NotCapable extends Error {}
```

**Example:**

```typescript
// Running without --allow-read
try {
  const data = await Deno.readFile("./file.txt");
} catch (error) {
  if (error instanceof Deno.errors.NotCapable) {
    console.error("Deno permission --allow-read not granted");
  }
}
```

**Common causes:**
- Missing `--allow-read` permission
- Missing `--allow-write` permission
- Missing `--allow-net` permission
- Missing `--allow-run` permission
- Missing `--allow-env` permission
- Missing `--allow-ffi` permission
- Missing `--allow-sys` permission

## I/O Errors

### InvalidData

Raised when an operation returns data that is invalid.

```typescript
class InvalidData extends Error {}
```

**Example:**

```typescript
try {
  const data = await Deno.readFile("corrupted.json");
  JSON.parse(new TextDecoder().decode(data));
} catch (error) {
  if (error instanceof Deno.errors.InvalidData) {
    console.error("File contains invalid data");
  }
}
```

### UnexpectedEof

Raised when attempting to read bytes from a resource, but EOF was unexpectedly encountered.

```typescript
class UnexpectedEof extends Error {}
```

**Example:**

```typescript
try {
  const file = await Deno.open("file.txt");
  const buffer = new Uint8Array(1000);
  const bytesRead = await file.read(buffer);
  if (bytesRead === null) {
    throw new Deno.errors.UnexpectedEof();
  }
} catch (error) {
  if (error instanceof Deno.errors.UnexpectedEof) {
    console.error("Unexpected end of file");
  }
}
```

### WriteZero

Raised when expecting to write to an I/O buffer resulted in zero bytes being written.

```typescript
class WriteZero extends Error {}
```

### WouldBlock

Raised when the OS would need to block but an asynchronous (non-blocking) API is used.

```typescript
class WouldBlock extends Error {}
```

### BrokenPipe

Raised when trying to write to a resource and a broken pipe error occurs.

```typescript
class BrokenPipe extends Error {}
```

**Example:**

```typescript
try {
  await Deno.stdout.write(encoder.encode("Hello\n"));
} catch (error) {
  if (error instanceof Deno.errors.BrokenPipe) {
    console.error("Pipe to stdout was closed");
  }
}
```

## Resource Errors

### BadResource

```typescript
class BadResource extends Error {}

```

### Busy

Raised when the underlying IO resource is being awaited on in another block of code.

```typescript
class Busy extends Error {}
```

## Error Handling Best Practices

### Specific Error Handling

```typescript
try {
  const file = await Deno.open("./data.json");
  const data = await file.readAll();
  file.close();
} catch (error) {
  if (error instanceof Deno.errors.NotFound) {
    console.error("File not found, creating default");
    await createDefaultFile();
  } else if (error instanceof Deno.errors.PermissionDenied) {
    console.error("Permission denied to read file");
  } else if (error instanceof Deno.errors.NotCapable) {
    console.error("Deno --allow-read permission not granted");
  } else {
    console.error("Unexpected error:", error);
    throw error;
  }
}
```

### Network Error Handling

```typescript
async function fetchWithRetry(url: string, retries = 3): Promise<Response> {
  for (let i = 0; i < retries; i++) {
    try {
      return await fetch(url);
    } catch (error) {
      if (error instanceof Deno.errors.ConnectionRefused) {
        console.log(`Connection refused, retry ${i + 1}/${retries}`);
        await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)));
      } else if (error instanceof Deno.errors.TimedOut) {
        console.log(`Request timed out, retry ${i + 1}/${retries}`);
      } else {
        throw error; // Don't retry other errors
      }
    }
  }
  throw new Error(`Failed after ${retries} retries`);
}
```

### File Operations with Fallback

```typescript
async function readConfigOrDefault(path: string): Promise<Config> {
  try {
    const data = await Deno.readTextFile(path);
    return JSON.parse(data);
  } catch (error) {
    if (error instanceof Deno.errors.NotFound) {
      console.log("Config file not found, using defaults");
      return getDefaultConfig();
    } else if (error instanceof Deno.errors.InvalidData) {
      console.error("Config file is corrupted, using defaults");
      return getDefaultConfig();
    } else {
      throw error;
    }
  }
}
```

### Resource Cleanup

```typescript
async function processFile(path: string) {
  const file = await Deno.open(path);
  
  try {
    const data = new Uint8Array(1024);
    const bytesRead = await file.read(data);
    
    if (bytesRead === null) {
      throw new Deno.errors.UnexpectedEof();
    }
    
    return data.slice(0, bytesRead);
  } catch (error) {
    if (error instanceof Deno.errors.BadResource) {
      console.error("File handle became invalid");
    }
    throw error;
  } finally {
    file.close(); // Always cleanup
  }
}

// Or use `using` for automatic cleanup
async function processFileAuto(path: string) {
  using file = await Deno.open(path);
  const data = new Uint8Array(1024);
  const bytesRead = await file.read(data);
  return bytesRead !== null ? data.slice(0, bytesRead) : new Uint8Array();
  // file is automatically closed
}
```

## Complete Error Handling Example

```typescript
import { assertEquals } from "jsr:@std/assert";

class FileManager {
  async readFile(path: string): Promise<string> {
    try {
      return await Deno.readTextFile(path);
    } catch (error) {
      if (error instanceof Deno.errors.NotFound) {
        throw new Error(`File not found: ${path}`);
      } else if (error instanceof Deno.errors.NotCapable) {
        throw new Error("Missing --allow-read permission");
      } else if (error instanceof Deno.errors.PermissionDenied) {
        throw new Error(`OS denied access to: ${path}`);
      } else if (error instanceof Deno.errors.IsADirectory) {
        throw new Error(`Path is a directory: ${path}`);
      } else {
        throw new Error(`Failed to read file: ${error.message}`);
      }
    }
  }

  async writeFile(path: string, content: string): Promise<void> {
    try {
      await Deno.writeTextFile(path, content);
    } catch (error) {
      if (error instanceof Deno.errors.NotCapable) {
        throw new Error("Missing --allow-write permission");
      } else if (error instanceof Deno.errors.PermissionDenied) {
        throw new Error(`OS denied write access to: ${path}`);
      } else if (error instanceof Deno.errors.IsADirectory) {
        throw new Error(`Cannot write to directory: ${path}`);
      } else {
        throw new Error(`Failed to write file: ${error.message}`);
      }
    }
  }

  async deleteFile(path: string): Promise<void> {
    try {
      await Deno.remove(path);
    } catch (error) {
      if (error instanceof Deno.errors.NotFound) {
        // File already doesn't exist, that's okay
        return;
      } else if (error instanceof Deno.errors.NotCapable) {
        throw new Error("Missing --allow-write permission");
      } else {
        throw new Error(`Failed to delete file: ${error.message}`);
      }
    }
  }
}
```

## Error Hierarchy

All Deno errors extend the base `Error` class:

```
Error
 └── Deno.errors.*
     ├── NotFound
     ├── PermissionDenied
     ├── NotCapable
     ├── ConnectionRefused
     ├── ConnectionReset
     ├── AlreadyExists
     ├── BadResource
     ├── Busy
     ├── TimedOut
     └── ... (and more)
```

This means you can catch all Deno-specific errors with:

```typescript
try {
  await someDenoOperation();
} catch (error) {
  if (error instanceof Error) {
    // Handle all errors including Deno errors
    console.error(error.message);
  }
}
```
